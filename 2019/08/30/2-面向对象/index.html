成员变量和局部变量的区别
A:位置不同：

    成员变量：在类中方法外

    局部变量：在方法内

B：在内部中位置不同

    成员变量：在堆内存中

    局部变量：在栈内存

C：生命周期不同：

    成员变量：随着对象的创建而存在，随着对象的消失而消失

    局部变量：随着方法调用而存在，随着方法的调用完毕而消失

D：初始化值不同：

    成员变量：有默认初始化值

    局部变量：没有默认初始化值，必须定义，赋值，然后才能使用

继承

多个类中存在相同属性和行为，将这些内容抽取到单独的一个类中

继承的优势：

1）：提高了代码的复用性

2）：提高了代码的维护性

3）：让类与类之间产生了关系，是多态的前提（类与类的耦合性太高）

开发的原则：低耦合、高内聚

耦合：类与类的关系

内聚：就是自己完成某个事件的能力
Java中继承的特点
A：Java只支持单继承，不支持多继承

B：支持==多层==继承（继承体系）
继承的注意事项
1）：子类只能继承父类所有的非私有成员变量（成员变量、成员方法）

2）：子类不能继承父类的构造方法，但是可以通过super关键字去访问父类构造方法

3）：不要为了部分功能去继承
继承中成员变量的关系

​ 子类和父类的成员变量一样时，就近原则

继承中构造方法的关系

子类的所有构造方法默认都会访问父类中==空参数==的构造方法

为什么？因为子类会继承父类的数据，如果父类的数据没有初始化，是不能访问的，所以父类的数据先初始化。

继承中成员方法的关系
子类中的方法和父类中的方法名重名        先找子，后找父类
方法重写
子类中出现了和父类中一摸一样的方法声明

应用：子类需要父类的功能，而功能主体子类有自己的特有内容，可以重写父类的方法

注意事项：

    父类的私有方法不能被重写

    子类重写父类方法时，访问权限不能更低，最好一致

由于继承方法中有方法重写的现象，所以父类的功能会被子类给覆盖掉，但是有些时候，我们不想让子类覆盖父类

的方法，只想让他使用，针对这种情况，Java提供了关键字final

final可以修饰类，方法、变量

final修饰类、方法、变量的特点

类：可以修饰类，但是该类不能被继承

方法：该方法不能被重写

变量：该变量不能被重新赋值，因为这个变量是常量
final修饰局部变量

​ 一般的权限修饰符，去修饰局部变量是没有任何意义的，但是final却有意义

​ 基本类型

​ 引用类型

final修饰变量的初始化时机

​ 1）：被final修饰的变量只能赋值一次

​ 2）：在构造方法完毕前（非静态变量）

抽象类

在Java中一个没有方法体的方法应该定义为抽象方法，而类中如果有抽象方法，该类必须定义为抽象类

抽象类的特点
1）：抽象类和抽象方法必须用abstract修饰

2）：抽象类中不一定有抽象方法，但是有抽象方法的类必须定义为抽象类

3）：抽象类不能实例化

4）：如果不想重写抽象方法，该子类是一个抽象类

    重写所有的抽象方法，这个时候子类是一个具体的类
抽象类的成员特点
成员变量

    既可以是变量，也可以是常量

构造方法

    有构造方法，用于父类的初始化

成员方法

    既可以是抽象的，也可以是非抽象的

抽象类的成员方法特性：

    A：抽象方法    强制要求子类做的事情

    B：非抽象方法    子类继承的事情，提高代码的复用性
抽象类的小问题
1 ）：一个类如果没有抽象方法，可以定义为抽象类吗？如果有，有什么意义

        A：可以

        B：不让外部对这个类创建对象

2）：abstract不能和哪些关键字共存

    private    冲突    final    冲突    static     无意义

接口

​ 为了体现事物的拓展性（狗跳火圈），Java使用接口来定义这些额外功能，并不给出具体实现

接口的特点
1）：接口用关键字interface表示

    interface  接口名  {}

2）：类实现接口用implements表示

    class 类名    implements  接口名  {}

3）：接口不能实例化

    要想实例化就通过多态来实现

4）：接口的子类

    A：可以是抽象类，但是没有意义

    B：可以是具体类，要重写接口中的所有方法（推荐方案）

总结现有的多态使用

具体类多态    （几乎不用）

抽象类多态    （常用）

接口多态        （最常用）
接口的特点
成员变量

    只能是常量，并且是静态的

    默认修饰符  public static final

构造方法

    接口没有构造方法

成员方法

    只能是抽象方法

    默认修饰符    public abstract

所有的类都默认继承自一个类：Object、类Object是类层次的根类

java中类与类，类与接口，接口与接口的关系

类与类：

继承关系，只能单继承，可以多层继承

类与接口：

实现关系，可以单实现，也可以多实现

并且在继承一个类的同时实现多个接口

接口与接口

继承关系，可以单继承，也可以多继承
抽象类与接口的区别
1）: 成员上的区别

    抽象类

        成员变量：可以常量，可以变量

        构造方法：有

        成员方法：可以抽象，可以非抽象

    接口

        成员变量：只能是常量

        成员方法：必须是抽象方法

2）：关系区别（见上）

3）：设计理念的区别

    抽象类：is a  定义的是该继承体系的共性功能

​ 接口：like a 定义的是该继承体系的拓展功能

形式参数和返回值问题

形式参数：
基本类型（不讲）

引用类型：

    类：需要的是该类的对象

    抽象类：需要的是抽象类的子类对象

    接口：需要的是该接口的实现类对象
返回值类型
引用类型：
    类：返回的是该类的对象

    抽象类：返回的是该抽象类的子类对象

    接口：返回的是接口的实现类对象

包

包就是文件夹

作用：把相同的类名放到不同的包中，对类进行分类管理

package  包名；

1）：package语句必须是程序第一条可执行的代码

2）：package语句在一个Java文件中只能有一个

内部类

​ 把类定义在其他类的内部，这个类就叫内部类

内部类的访问特点
（1）内部类可以直接访问外部类的成员，包括私有

（2）外部类要访问内部类的成员，必须创建对象
内部类的位置
（1）在成员位置定义：成员内部类        成员内部类成员的直接访问

    A：成员内部类的修饰符

        private    使数据更加安全

        static    为了访问数据方便

        注意：静态内部类访问的外部数据必须用静态修饰
（2）在局部位置定义：局部内部类

    （1）可以直接访问外部类的成员

    （2）在局部位置，可以创建内部类的对象，通过对象调用内部类的 方法，来使用局部内部类的功能
（3）匿名内部类（在局部内部类的基础上）

    就是内部类的简化写法

    前提：存在一个接口或者类，可以是具体类，也可以是抽象类
